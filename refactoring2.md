# 🔧 すごいAI道場サイトをフルリファクタリングしたいので、以下の作業をプロジェクト全体に対して行ってください。

## 🎯 現状の問題と目的
- 各HTMLにヘッダーやCSSが重複しており、編集コストが高い
- CSSが各ファイルの `<style>` にベタ書きされていて管理が困難
- JavaScriptが各HTMLごとに散らばっていて挙動が不統一
- カテゴリごとに色を変える必要があるが個別CSSに依存してしまっている

これらを **1つの共通構造** にまとめたいです。

---

# ✅ 目的：以下を達成する完全リファクタリング
1. **ヘッダーを header.html に切り出し、全ページで共通化する**
2. **CSS を共通化し、カテゴリ別に追加CSSで上書きできる構造にする**
3. **JavaScript の共通部分を common.js に集約する**
4. **既存ページは、この新しい構造を参照するように改修する**

---

# 🗂 フォルダ構成（これに整える）

```
/
├── header.html              ← 共通ヘッダー
├── common.js                ← 共通JS（ハンバーガー等）
├── styles/
│   ├── base.css             ← 全ページ共通ベースCSS
│   ├── nav.css              ← ヘッダー/ナビ専用CSS
│   ├── lesson.css           ← レッスンページ共通CSS
│   ├── category_ai.css      ← AI一般（オレンジ）
│   ├── category_startup.css ← 起業・ビジコン（青）
│   ├── category_creative.css← クリエイティブ（緑）
│   └── category_community.css← コミュニティ（紫）
│   └── category_tutorial.css← チュートリアル（灰）
└── （既存HTMLファイル）
```

---

# 📝 詳細指示（実際の作業内容）

## ① ヘッダーの共通化
- 各HTMLにある `<nav class="main-nav"> 〜 </nav>` を抽出して `header.html` として配置
- 各HTMLの `<body>` 直下に：

```html
<div id="site-header"></div>
```

を配置し、

```html
<script src="common.js"></script>
```

の読み込みで `header.html` を注入する形式に統一。

---

## ② CSS の共通化
- 各ページの `<style>` タグを調べ、  
  **共通部分は base.css / nav.css / lesson.css に分割してまとめる**
- 以下の仕組みを作る：

### ✔ 全ページは基本的にこれだけ読み込む

```html
<link rel="stylesheet" href="styles/base.css">
<link rel="stylesheet" href="styles/nav.css">
<link rel="stylesheet" href="styles/lesson.css">
```

### ✔ カテゴリ別ページでは追加CSSを読み込む

例：AI一般カテゴリ（オレンジ）の場合

```html
<link rel="stylesheet" href="styles/category_ai.css">
```

この category 系CSS は、  
**色・アクセントカラー・ボタンテーマ** の上書き専用。

---

## ③ JS の共通化（common.js）
- 全ページで使われている「ハンバーガーメニュー」「ナビの開閉」などを共通化
- YouTube Player 初期化関数なども共通化できる部分はここに集約
- ページ固有JSは残すが、重複機能は削除する

---

## ④ 既存HTMLの書き換え
- `<style>` を全て削除し、代わりに上記CSSファイルを読み込むよう編集
- `<nav>` は削除し、代わりに `<div id="site-header"></div>` に置き換え
- `<script>` のうち共通部分は排除して common.js に移動

---

# 📌 出力形式の指示
作業は以下の順番で進めてください：

1. まず、プロジェクト内のすべてのHTMLファイルの現状構造を解析し、  
   「共通化すべきCSS一覧」「カテゴリ別の違い一覧」「共通JS一覧」を作成  
2. 提案するフォルダ構成に沿って、新しいファイルを生成  
3. 各HTMLを新構造へリファクタリング  
4. すべての変更内容をサマリとして最後にまとめる  
5. もしランタイム上限で中断したら、次のメッセージで「続き」と言えば再開できるように  
   作業の進捗をログとして保持すること

# 🟧 注意事項
- 既存のデザインやレイアウトはできる限り変更せず「構造だけを改善」してください
- 色設定はカテゴリCSSで確実に上書きできる状態にブレイクダウンしてください
- 動作確認が必要な箇所（navメニューなど）は、JSの動作コードも修正
