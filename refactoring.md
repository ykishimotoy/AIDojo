# ゴール
このリポジトリ（すごいAI道場のサイト）の HTML / CSS をリファクタして、
- 共通レイアウト・共通コンポーネントの CSS を styles.css に集約
- 各 HTML にベタ書きされている <style> ブロックを削除
- 今後ページが増えても「1箇所を直せば全体に反映される」構造にする
- 既存デザイン（色・余白・サイズ・レスポンシブ挙動）は基本的に変えない
ことを目指してください。

特に、今は各 HTML ファイルの <head> 内に大きな <style> ブロックがあり、
同じクラス名・同じようなレイアウトがファイルごとに重複定義されている状態です。
これを整理して「共通CSS＋カテゴリ別カラーバリエーション」という構造にしたいです。

基準となるヘッダーの項目一覧は、test-badge-system.htmlを参照してください。

---

# 現状イメージ（例）
例として、tutorial.html には下記のような構造があります：

- <head> 内に styles.css の読み込み + 巨大な <style> ブロック
- .lesson-page, .lesson-header, .lesson-title-row, .lesson-number-badge, .content-section,
  .navigation-buttons, .nav-btn などのクラスが定義されている
- 他のレッスンページや HTML ファイルにも、名前や役割がほぼ同じクラスが
  別々に <style> で書かれているはず

さらに、同じような UI パーツでも「カテゴリごとに色だけ違う」ケースがあります。
（例：ビジコンカテゴリなら青、基礎カテゴリならオレンジ、など）

# やってほしいこと（ステップ）

## 1. 全 HTML / CSS ファイルのスキャンと構造把握
- リポジトリ内の全ての .html と .css ファイルを読み込み、
  - 共通で使われているクラス
  - ページごとに微妙に違うが、本質的には同じ役割のクラス
  を洗い出してください。
- 特にレッスン系ページ（howtoAI/ 配下など）と
  共通レイアウト（ヘッダー・フッター・ナビゲーション）を重点的に見てください。

## 2. 共通・再利用可能なスタイルの抽出
- 以下のような「構造・レイアウト・タイポグラフィ」に関する CSS を、
  styles.css の共通クラスとして定義しなおしてください：
  - コンテナ系: .lesson-page, .lesson-header, .lesson-title-row, .content-section, .navigation-buttons など
  - テキスト系: .lesson-title, .lesson-description, 見出し h2/h3 のスタイル
  - ボタン系: .nav-btn, .nav-btn.prev, .nav-btn.next など
  - レスポンシブ対応: @media (max-width: ...) の中身も、できる限り共通化
- 上記の「構造・レイアウト・フォントサイズ・余白・ボーダー・角丸・影」などは
  可能な限り styles.css に一元化し、
  各 HTML の <style> 内からは削除してください。

## 3. カテゴリ別「色違い」パターンの扱い
このリポジトリでは、
- 「同じパーツだが、カテゴリごとに色だけ違う」
というケースが存在します。

例：
- ビジコンカテゴリ → 青
- 基礎カテゴリ → オレンジ
- 他カテゴリも今後増える可能性あり

この場合は、以下の方針でリファクタしてください：

### 3-1. ベースクラスとバリアントクラスに分離
- まず「構造・余白・角丸・フォントサイズ」などを定義する**ベースクラス**を作成：
  - 例: .lesson-number-badge, .nav-btn など
- そのうえで、「色に関わるプロパティ（背景色・文字色・ボーダー色など）」は
  別の **カテゴリ別バリアントクラス** や **修飾クラス** に切り出してください。

例（あくまで一例、実際のカテゴリ名や色は既存定義に合わせてください）：

```css
/* ベース */
.lesson-number-badge {
  padding: 10px 20px;
  border-radius: 25px;
  font-weight: bold;
  font-size: 1.1rem;
  flex-shrink: 0;
}

/* カテゴリ別バリアント */
.lesson-number-badge--basic {
  background: #f46434;  /* 既存の基礎カテゴリの色を流用 */
  color: #fff;
}

.lesson-number-badge--startup {
  background: #4a90e2;  /* 既存のビジコン/スタートアップカテゴリの色 */
  color: #fff;
}

/* 他のカテゴリも同様に追加 */

*  もし既存の HTML で、同じクラス名なのにページごとに色だけ違う場合は、  
   その差分をこのような「バリアントクラス」に置き換えてください。

*  バリアントの付け方は BEM 風 (`block--modifier`) でも、  
   `.category-startup .lesson-number-badge` のように親にカテゴリクラスをつけても構いません。  
   どちらを採用するかは、全体構造を見て最も整合的な方法を選んでください。

---

### 3-2. カテゴリ情報の付与（必要に応じて）

*  ページによってカテゴリが明確に分かれている場合は、  
   `<body>` 要素または主要なラッパー要素にカテゴリを示すクラスを追加しても構いません：

   例：
   ```html
   <body class="category-basic">
   ...
   <body class="category-startup">

そのうえで CSS 側で：
.category-basic .lesson-number-badge { background: #f46434; ... }
.category-startup .lesson-number-badge { background: #4a90e2; ... }

*   のように書いてもOKです。

*   いずれの方法でも構いませんが、  
    「構造と色が混ざらない」「ベース＋バリアントという構造が明確になる」  
    ことを優先してください。

---

## 4. 各 HTML の <style> ブロックの削除と外部 CSS 化

*   共通化が完了したら、各 HTML ファイルの <head> 内にある  
    `<style> ... </style>` ブロックから、  
    すでに styles.css に移したルールを削除してください。

*   まだ styles.css に移していない「そのページ専用の、どうしても特殊なスタイル」がある場合は、  
    そのページ専用のクラス名になるように整理したうえで、  
    可能ならそれも styles.css か、別の `page-specific.css` のようなファイルに切り出してください。

*   最終的には可能な限り：

    -   HTML側には `<link rel="stylesheet" href="styles.css">` だけ
    -   レイアウトやコンポーネントのロジックはすべて CSS ファイル群に集約

    という形を目指してください。

---

## 5. 動作・見た目の保持

*   レイアウト崩れや、フォントサイズ・余白・レスポンシブ挙動が変わらないように注意してください。

*   色の違いについては、「カテゴリごとの差分」は必ず保持しつつ、  
    その定義方法だけを「ベース＋バリアント」構造に変える、というイメージです。

*   もしどうしても微妙に見た目が変わる可能性がある場合は、  
    その箇所と理由をコメントで明示してください。

---

## 6. 最後にやってほしいアウトプット

リファクタが完了したら：

1.  変更ファイル一覧（HTML / CSS）と、主な変更内容を箇条書きでまとめてください。

2.  代表的なコンポーネント（例：.lesson-page, .lesson-number-badge, .nav-btn など）について  
    -   旧状態（どのファイルにバラバラに書かれていたか）  
    -   新状態（styles.css 上のどのクラスが担当しているか）  
        を説明してください。

3.  カテゴリ別色分けをどう表現したか  
    （修飾クラス方式か、bodyのカテゴリクラス方式か）  
    を説明してください。

# 補足

*   途中で「このクラス設計のほうが良い」という案があれば、積極的に提案して構いません。

*   ただし、デザインを勝手に変えるのではなく、  
    **既存の見た目を保ったまま CSS の構造だけを整理する**  
    ことを最優先してください。

*   必要に応じて、小さなユーティリティクラス  
    （例：.text-muted, .btn-primary, .btn-secondary）  
    を導入しても構いません。
